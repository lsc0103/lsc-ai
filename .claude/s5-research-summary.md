## 三、船舶行业文档痛点

### 3.1 核心文档类型

| 文档类型 | 数量级 | 当前处理方式 | IDP 价值 |
|---------|--------|------------|---------|
| 出入涂清单/工程清单 | 单船 200-500 页 | 手工录入 Excel | 极高 |
| 检验报告（NDT/涂装/船级社）| 单船 2000+ 份 | 手工归档+录入 | 极高 |
| 合同/补充协议 | 50-100 份/年 | 律师人工审查 | 高 |
| 工艺卡片/施工记录 | 大量 | 纸质存档 | 高 |
| 设计变更通知（ECN/DCN）| 单船 200-500 份 | 手工流转 | 高 |
| 手写尺寸单据 | 大量 | 人工抄录到系统 | 中 |

### 3.2 文档管理的行业痛点

| 痛点 | 影响 |
|------|------|
| **手写体多** | 现场记录、检验签字大量手写，OCR 识别困难 |
| **格式不统一** | 扫描件/电子版/CAD图纸/Excel 混合 |
| **多语言** | 中/英/日/韩混合（DACKS 中日合资背景）|
| **审批链长** | 编制→校对→审核→批准→船级社→船东 |
| **合规严格** | 船级社（CCS/DNV/LR/ABS）+ IMO 要求文档完整追溯 |
| **版本混乱** | 设计变更频繁，文档版本管理困难 |

## 四、S5 IDP 五大场景

### 场景优先级

| 优先级 | 场景 | ROI | 技术难度 | 理由 |
|--------|------|-----|---------|------|
| **P0** | A — 出入涂清单 | 16-24万/年 | ★★★ | 数据量大、技术成熟、用户痛点强 |
| **P0** | D — 检验报告 | 20-30万/年 | ★★★ | 合规刚需、标准化程度高 |
| **P1** | C — 合同审查 | 15-25万/年 + 风险规避 | ★★★★ | 价值极高但需领域知识 |
| **P1** | E — 设计变更 | 10-20万/年 | ★★★★ | 全流程影响大 |
| **P2** | B — 手写单据 | 18-30万/年 | ★★★★★ | 技术挑战最大 |

### 场景 A：出入涂清单（P0）

- **输入**：PDF 扫描件/电子版，中英混合，多层嵌套表格
- **处理**：版面分析 → OCR → 表格结构化 → LLM 校验 → 结构化输出
- **输出**：JSON/Excel 标准格式，可推送 ERP/MES
- **ROI**：单船清单处理 40-60 人·天 → 8-12 人·天（节省 70-80%）

### 场景 D：检验报告（P0）

- **输入**：NDT 报告（RT/UT/MT/PT 四种无损检测）、涂装检查、船级社报告
- **处理**：文档分类 → OCR → 结构化提取 → 质量校验 → 归档
- **输出**：标准化数据入质量管理系统，支持追溯查询
- **ROI**：NDT 报告录入 15-20 分钟 → 2-3 分钟（节省 85%）

### 场景 C：合同审查（P1）

- **特殊性**：SAJ/BIMCO 标准、多方多币种、海事法律、长周期（18-36 月）
- **关键要素**：30+ 抽取字段（船型/价格/交付/技术规格/违约/保函等）
- **三级风险规则**：红色（延迟罚金>5%）、橙色（付款不均）、黄色（措辞模糊）
- **ROI**：单份审查 3-5 天 → 2-4 小时（节省 85-95%），风险遗漏率 10-15% → <2%

### 综合 ROI

| 分类 | 年节省金额（10 船规模） |
|------|---------------------|
| 场景 A：清单处理 | 16-24 万元 |
| 场景 B：手写单据 | 18-30 万元 |
| 场景 C：合同审查 | 15-25 万元 + 风险规避 |
| 场景 D：检验报告 | 20-30 万元 |
| 场景 E：设计变更 | 10-20 万元 |
| **合计直接节省** | **79-129 万元/年** |
| **间接价值** | **估计 2-5 倍于直接节省** |

## 五、技术选型确认

### 5.1 PaddleOCR v5 — 最优选型

| 维度 | PaddleOCR v5 | Surya | MinerU2.5 | Qwen3-VL |
|------|-------------|-------|-----------|----------|
| 中文印刷体 | **99.2%** | 98.1% | 98.9% | 98.8% |
| 中文手写体 | **94.7%** | 91.2% | 92.1% | 92.3% |
| 表格识别 | **96.5%** | 97.7% | 95.9% | 95.8% |
| 竖排文本 | **95.4%** | 88.2% | 90.1% | 91.3% |
| 推理速度 | **0.8s** | 1.5s | 1.8s | 4.5s |
| 离线部署 | **完全支持** | 支持 | 支持 | 需 vLLM |
| 开源协议 | Apache 2.0 | GPL 3.0 | Apache 2.0 | 自定义 |

**结论**：PaddleOCR 在中文全场景综合领先，且完全开源、支持国产硬件、离线部署。

### 5.2 成本分析

| 方案 | 5 年总成本 | 离线部署 | 可定制性 |
|------|-----------|---------|---------|
| **PaddleOCR（推荐）** | **¥28.8 万** | 完全支持 | 极高 |
| TextIn SaaS | ¥25-75 万 | 不支持 | 中 |
| 达观 IDPS | ¥30-100 万 | 不支持 | 中 |
| 百度智能文档 | ¥15-50 万 | 支持 | 高 |

### 5.3 船舶行业特殊技术挑战

| 挑战 | 解决方案 |
|------|---------|
| A0/A1 大幅面图纸 | 分块处理 + PaddleOCR 动态分辨率 |
| CAD/DWG 文字提取 | 优先矢量解析，备选 DWG→PNG→OCR |
| 中/英/日/韩混合 | PP-OCRv5 原生多语言支持 |
| 嵌套/跨页表格 | PP-StructureV3 + LLM 辅助合并 |
| 手写体 | 收集船厂样本微调模型 |
| 老旧文档（泛黄/模糊）| OpenCV 预处理管道（二值化/去噪/锐化/超分）|

## 六、UI/UX 设计指导

### 6.1 设计基调

| 维度 | 规范 |
|------|------|
| **主色** | 中远海运品牌蓝（PANTONE 2945）|
| **辅色** | 中远海运红（PANTONE 1795）— 告警/强调 |
| **背景** | 深色主题 #141414~#1f1f1f（降低长时注视疲劳）|
| **信息密度** | 高密度紧凑布局，表格为核心 |
| **操作效率** | 常用操作两次点击内到达 |
| **字号** | 最小 14px（老龄化用户群体）|
| **对比度** | ≥4.5:1（WCAG AA 标准）|

### 6.2 IDP 核心页面设计

#### OCR 结果展示 — 左右分栏同步

```
┌────────────────────────────────────────────────┐
│  工具栏：缩放 | 旋转 | 对照模式 | 导出           │
├──────────────────────┬─────────────────────────┤
│  原始文档图像         │  识别结果文本            │
│  (Bounding Box 标注)  │  (可编辑、逐行对应)      │
│                      │                         │
│  置信度色彩编码：      │  字段提取结果表格         │
│  绿>95% 黄80-95%     │                         │
│  红<80%              │                         │
├──────────────────────┴─────────────────────────┤
│  底部：置信度评分 | 修正计数 | 处理时间           │
└────────────────────────────────────────────────┘
```

#### 合同审查结果 — 风险三级标注

```
┌────────────────────────────────────────────────┐
│  合同全文 + 要素高亮                             │
│  蓝=主体 绿=金额 黄=期限 红=风险条款             │
├────────────────────────────────────────────────┤
│  右侧面板：                                     │
│  ├── 合同基本信息摘要                            │
│  ├── 风险评估（红/黄/绿仪表盘）                  │
│  ├── 条款差异（vs 标准模板 Diff View）            │
│  └── 审查意见（AI 建议 + 人工批注）              │
└────────────────────────────────────────────────┘
```

#### 批量文档处理 — 队列管理

```
总进度条 + 预计剩余时间
每文件独立状态（成功/处理中/排队/失败）
失败文件提供重试按钮 + 具体错误
支持取消排队中的任务
```

### 6.3 Workbench 集成方式

IDP 结果通过 Workbench 展示，复用现有 Chat + 右侧面板架构：
- 对话中上传文档 → AI 自动调用 IDP 工具 → Workbench 展示结果
- 对话驱动（"分析这份合同"）和功能驱动（直接上传到 IDP 页面）两种入口
- OCR 结果用 DataTable 组件展示
- 合同要素用 Workbench 自定义 Schema 展示

### 6.4 信创合规

- 前端兼容信创浏览器（基于 Chromium，React+AntD 无需特殊适配）
- 字体使用思源黑体/宋体（开源中文字体，无版权风险）
- 所有数据展示界面标注数据来源和更新时间
- 审计日志（S4.5 已实现）满足央企审计要求

## 八、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| PaddleOCR 在 Windows Docker 兼容性 | 中 | 中 | WSL2 + Docker Desktop 已验证可行 |
| 手写体识别准确率不达标 | 中 | 低 | 手写场景放 P2，先做印刷体场景 |

| 真实船舶文档测试样本缺乏 | 高 | 高 | 使用公开船舶文档模板 + 用户提供样本 |
