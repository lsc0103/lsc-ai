## 一、行业背景与趋势

### 1.1 全球船舶制造 AI 转型现状

2024-2025 年，全球船舶制造业正经历深刻的 AI 驱动转型：

- **中国船厂**在 2024 年拿下全球 74% 的造船吨位订单，产能压力倒逼数字化升级
- **韩国三大船厂**（HD 现代重工、三星重工、韩华海洋）全面推进"智能船厂"战略，目标 2030 年前实现生产效率提升 30%
- **AI 在船舶全生命周期的渗透率**：设计阶段最高（约 40%），建造阶段次之（约 25%），文档处理最低（约 10%），是最大的提效空间

### 1.2 IDP 市场趋势

- 全球 IDP 市场规模 2024 年达 23 亿美元，预计 2030 年增长至 123.5 亿美元（CAGR 33.1%）
- **制造业**是增长最快的 IDP 应用领域，增速达 24.5%
- 制造企业使用 IDP 后平均削减 30-50% 的文档处理成本
- GenAI（生成式 AI）正在重塑 IDP：从模板匹配进化为上下文理解，大幅降低部署门槛

### 1.3 大连中远川崎（DACKS）数字化基础

- **2023 年**：入选工信部"智能制造示范工厂揭榜单位"
- **数字化战略**：围绕"绿色化生产、数字化运营、智能化制造"三条主线
- **已有能力**：一体化三维数字化设计平台、工业互联网平台、移动应用平台、供应链管理平台
- **智能报价系统**：已实现散货船报价单自动生成，报价效率提升一倍
- **数字化运营平台**（一期）：2024 年 6 月上线，实现数据整合与共享

## 三、IDP 五大核心场景深入分析

### 场景 A：出入涂清单/工程清单（PDF 表格提取）

#### A.1 文档特征

| 特征 | 描述 |
|------|------|
| **文档格式** | PDF（扫描件 + 电子版混合），偶有 Excel/Word |
| **表格结构** | 多层嵌套表头、合并单元格、跨页表格 |
| **语言** | 中英混合（船体部位英文编号 + 中文描述） |
| **编号体系** | 分段号（如 S1-01-P）、涂装区域码、材料编号 |
| **数据量** | 单船 200-500 页，涵盖 5000-20000 个涂装区域 |
| **质量** | 扫描件可能有倾斜、模糊、盖章遮挡 |

#### A.2 典型字段

```
分段号 | 涂装区域 | 面积(m2) | 涂料型号 | 涂层数 | 膜厚要求(μm) | 表面处理 | 备注
S1-01-P | 外板水线上 | 245.6 | SIGMA-EP | 3 | 250 | Sa2.5 | 含锌底漆
```

#### A.3 自动化处理流程

```
PDF 文件上传
    │
    ▼
[1] 版面分析（Layout Analysis）
    ├─ 识别表格区域 vs 文字区域
    ├─ 检测表头结构（多层嵌套）
    └─ 处理跨页表格合并
    │
    ▼
[2] OCR 文字识别（PaddleOCR）
    ├─ 中英混合识别
    ├─ 数字/编号精确识别
    └─ 倾斜校正 + 去噪
    │
    ▼
[3] 表格结构化（Table Extraction）
    ├─ 单元格定位与内容填充
    ├─ 合并单元格处理
    └─ 表头-数据行关联
    │
    ▼
[4] LLM 智能校验（DeepSeek）
    ├─ 编号格式验证（正则 + 规则）
    ├─ 涂料型号标准化（映射表）
    ├─ 面积数值合理性检查
    └─ 异常标注与人工确认
    │
    ▼
[5] 结构化输出
    ├─ JSON/Excel 标准格式
    ├─ 推送至 ERP/MES 系统
    └─ 差异报告（与计划对比）
```

#### A.4 下游系统对接

| 系统 | 对接方式 | 数据用途 |
|------|---------|---------|
| ERP（SAP/用友） | API/中间表 | 物料需求计划、成本核算 |
| MES 生产执行系统 | 消息队列 | 涂装工序排程、进度跟踪 |
| 质量管理系统 | REST API | 涂装检查记录关联 |
| 项目管理系统 | 文件同步 | 进度节点文件归档 |

#### A.5 技术挑战与解决方案

| 挑战 | 解决方案 |
|------|---------|
| 扫描件质量差 | 图像预处理（去噪、二值化、倾斜校正）|
| 跨页表格 | 表头特征匹配 + 页间连接算法 |
| 合并单元格 | 基于线条检测的单元格合并推理 |
| 中英混合编号 | 双模型融合识别 + 后处理规则 |

### 场景 B：工程物料尺寸单据（手写体 OCR）

#### B.1 船舶现场手写单据特点

| 特征 | 描述 |
|------|------|
| **书写环境** | 船舱内、甲板上，光线差、振动、潮湿 |
| **纸张质量** | 普通复写纸、工艺卡片，可能有油污/水渍 |
| **书写风格** | 多人书写风格各异，数字 1/7、0/6 易混淆 |
| **内容类型** | 尺寸数据（mm）、管件型号、焊工号、日期 |
| **结构化程度** | 半结构化（印刷表格 + 手写填充） |

#### B.2 典型手写单据字段

```
管件编号: P-3201-50A-S    (印刷+手写)
实测外径: 60.3 mm         (手写)
实测壁厚: 5.5 mm          (手写)
焊工编号: W-2024-0156     (手写)
检验日期: 2026/02/10      (手写)
检验员签名: [签名]         (手写)
```

#### B.3 手写体识别技术挑战

| 挑战 | 难度 | 解决方案 |
|------|------|---------|
| 字迹潦草 | ★★★★★ | 多模型投票（PaddleOCR + LLM 视觉模型）|
| 数字精度要求高 | ★★★★ | 尺寸范围校验 + 上下文约束推理 |
| 纸面污损 | ★★★ | 图像增强（对比度调整、去噪、阴影消除）|
| 多人笔迹 | ★★★★ | 笔迹分段 + 分别识别 |
| 印刷+手写混排 | ★★★ | 区域分割（印刷区域 vs 手写区域）|

#### B.4 推荐技术方案

```
手写单据扫描/拍照
    │
    ▼
[1] 图像预处理
    ├─ 透视校正（四角检测）
    ├─ 去油污/水渍（图像修复）
    └─ 增强对比度
    │
    ▼
[2] 区域分割
    ├─ 印刷模板匹配（检测印刷字段名）
    └─ 手写区域定位（填写区域）
    │
    ▼
[3] 手写识别
    ├─ PaddleOCR 手写模型（基础识别）
    ├─ Transformer OCR（增强识别）
    └─ LLM 多模态视觉（Qwen-VL / GPT-4o 兜底校验）
    │
    ▼
[4] 后处理校验
    ├─ 尺寸值范围验证（管径 10-2000mm）
    ├─ 编号格式正则校验
    ├─ 日期合理性检查
    └─ 交叉验证（外径 vs 壁厚合理性）
    │
    ▼
[5] 人工复核
    ├─ 低置信度字段高亮
    ├─ 原图对照显示
    └─ 一键确认/修正
```

#### B.5 与 ERP/MES 系统对接

| 对接场景 | 数据流向 | 频率 |
|---------|---------|------|
| 来料检验 → ERP | 实测尺寸 → 物料主数据 | 实时 |

|------|------------|------|
| 清晰手写数字 | 95-98% | 97%+ |

| **变更条款** | 设计变更审批流程、价格调整机制 | ★★★★ |

#### C.3 风险检测规则

```yaml
# 一级风险（红色告警）
- 延迟交付罚金率超过 5%/周或无上限
- 退船条件过于宽泛（低于行业标准 150-180 天）
- 退款保函无到期日或自动延期条款
- 技术规格引用的规范版本已过期
- 缺少不可抗力条款或定义过窄

# 二级风险（橙色警告）
- 付款节点分布不均（前期过多或过少）
- 汇率风险未约定调整机制
- 保险条款缺失或覆盖不足
- 争议解决条款不明确
- 变更价格调整机制缺失

# 三级风险（黄色提示）
- 格式或措辞与标准模板差异大
- 关键条款使用模糊表述（"合理""适当"）
- 交叉引用不一致
- 附件缺失或版本不匹配
```

#### C.4 技术实现方案

```
合同 PDF/Word 上传
    │
    ▼
[1] 文档解析
    ├─ PDF 文本提取（电子版直接提取）
    ├─ 扫描件 OCR（PaddleOCR）
    └─ 结构化：目录→章节→条款
    │
    ▼
[2] 要素抽取（LLM + NER）
    ├─ Prompt Engineering：定义 30+ 要素模板
    ├─ 命名实体识别：船名/日期/金额/机构
    └─ 表格信息提取：付款计划/技术参数表
    │
    ▼
[3] 风险分析
    ├─ 规则引擎：30+ 预定义风险规则
    ├─ LLM 推理：条款语义分析
    ├─ 知识图谱：与标准合同对比
    └─ 版本比对：标记与上版差异
    │
    ▼
[4] 审查报告生成
    ├─ 要素汇总表
    ├─ 风险清单（按等级排序）
    ├─ 修改建议
    └─ 可追溯：每条结论链接原文位置
```

### 场景 D：检验报告/质量记录（标准化提取）

#### D.1 NDT 报告（四种无损检测）

| 检测方法 | 英文缩写 | 报告特征 | IDP 处理重点 |
|---------|---------|---------|-------------|
| 射线检测 | RT | 底片编号、缺陷等级、焊缝位置图 | 图像+文字混合提取 |
| 超声波检测 | UT | 波形数据、探伤灵敏度、缺陷定位 | 数值精确提取 |
| 磁粉检测 | MT | 磁化方式、缺陷描述、长度测量 | 半结构化文本抽取 |
| 渗透检测 | PT | 渗透剂型号、显示痕迹描述、评定结果 | 分类判定 |

**通用字段**：
```
检测编号 | 工件编号 | 焊缝号 | 检测日期 | 检测标准
灵敏度/级别 | 缺陷描述 | 缺陷位置 | 质量等级 | 评定结论
检测员 | 审核员 | 船级社验船师
```

#### D.2 涂装检查记录

| 检查项目 | 数据类型 | 提取难度 |
|---------|---------|---------|
| 表面处理等级（Sa2.5 等） | 标准代码 | ★★ |
| 环境条件（温度/湿度/露点） | 数值 + 单位 | ★★ |
| 干膜厚度（DFT）测量值 | 多点数值 | ★★★ |
| 涂料批号和有效期 | 编号 + 日期 | ★★ |
| 检查结论 | 合格/不合格/有条件接受 | ★★ |

#### D.3 船级社检验报告

| 报告类型 | 典型内容 | IDP 价值 |
|---------|---------|---------|
| 建造检验报告 | 结构/焊接/涂装/设备检验结论 | 关键节点合规确认 |
| 入级检验报告 | 入级条件、限制条件、后续要求 | 合规追踪 |
| 年度/特别检验报告 | 发现问题清单、整改要求、期限 | 整改任务生成 |

#### D.4 标准化提取流程

```
检验报告上传（PDF/扫描件/照片）
    │
    ▼
[1] 文档分类
    ├─ 自动识别报告类型（RT/UT/MT/PT/涂装/船检）
    └─ 匹配对应的提取模板
    │
    ▼
[2] OCR + 结构化提取
    ├─ 固定字段：表格区域定位 → 键值对提取
    ├─ 自由文本：缺陷描述 → NLP 实体抽取
    └─ 签名区域：检测员/审核员识别
    │
    ▼
[3] 质量规则校验
    ├─ 缺陷等级与接受标准比对
    ├─ 膜厚值与规范要求比对
    ├─ 检测覆盖率计算（已检/应检）
    └─ 超标项自动标红
    │
    ▼
[4] 输出与归档
    ├─ 结构化数据入库
    ├─ 关联到焊缝/分段/船号
    ├─ 统计报表（合格率/缺陷分布）
    └─ 船级社送审包自动生成
```

#### D.5 与质量管理系统集成

- **自动关联**：报告中的焊缝号自动关联到生产管理系统中的焊接记录
- **超标告警**：发现不合格项时自动生成 NCR（不合格品报告）并推送至相关人员
- **统计分析**：按分段/焊工/时间段统计缺陷率，生成质量趋势报告
- **合规追踪**：船级社要求的整改项自动生成任务，跟踪至闭环

### 场景 E：设计变更通知（ECN/DCN）

#### E.1 船舶设计变更的文档体系

```
设计变更请求（ECR）
    │ 发起方：设计/生产/船东/船级社
    ▼
设计变更通知（ECN/DCN）
    │ 内容：变更描述、原因、影响范围、图纸清单
    ▼
变更图纸（修订版）
    │ 新版 CAD 图纸、标注变更区域
    ▼
影响评估报告
    │ BOM 变化、工时影响、成本影响、进度影响
    ▼
审批记录
    │ 设计→工艺→生产→质量→船东→船级社
    ▼
闭环确认
    实施确认 + 验证记录
```

#### E.2 ECN 文档的 IDP 处理需求

| 处理需求 | 技术方案 | 价值 |
|---------|---------|------|
| **变更要素抽取** | NLP 从 ECN 文本提取：变更编号/原因/影响分段/涉及图纸 | 自动建立变更台账 |
| **图纸版本对比** | CV 图像对比 + CAD 模型比较 | 快速定位变更区域 |
| **BOM 影响分析** | 表格对比（旧 BOM vs 新 BOM） | 自动计算物料差异 |
| **审批状态追踪** | OCR 签名识别 + 流程引擎 | 防止未审批变更流入生产 |
| **下游通知** | 自动解析影响范围 → 推送至相关部门 | 避免信息遗漏 |

#### E.3 版本对比与影响分析

```
ECN 文档上传
    │
    ▼
[1] 变更要素抽取
    ├─ 变更编号、日期、发起部门
    ├─ 变更原因（分类：设计优化/规范更新/船东要求/生产适配）
    ├─ 影响范围（分段号、系统、区域）
    └─ 涉及图纸清单（图号 + 版本号）
    │
    ▼
[2] 图纸版本对比
    ├─ 获取新旧版本图纸
    ├─ 图层差异检测（结构/管路/电气）
    ├─ 标注变更区域（红色高亮）
    └─ 生成差异报告
    │
    ▼
[3] 影响分析
    ├─ BOM 变化：增/删/改物料自动计算
    ├─ 工时影响：基于历史数据估算
    ├─ 成本影响：物料+工时→成本增减
    ├─ 进度影响：关联生产排程评估延期风险
    └─ 关联变更：检测是否影响其他 ECN
    │
    ▼
[4] 自动通知与追踪
    ├─ 按影响范围自动推送至相关部门
    ├─ 审批流程自动发起
    ├─ 实施进度跟踪
    └─ 闭环确认（实施+验证）
```
