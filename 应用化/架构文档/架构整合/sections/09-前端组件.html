一、Workbench 组件库概述

1.1 核心理念

LSC-AI 的 Workbench 需要支持 AI 动态构建界面。AI 可以根据用户需求，调用预设的组件库来构建可视化内容。

AI 动态构建流程
1. 用户请求

"帮我查询今天 WMS 的出入库流水并生成报表"

2. AI 决策
  • 调用 WMS API 查询数据
  • 使用 DataTable 组件展示明细
  • 使用 BarChart 组件展示统计
  • 提供 ExportButton 导出
3. Workbench 渲染

动态渲染组件 Schema，生成可视化报表界面

1.2 设计原则

原则说明
声明式AI 只需描述要展示什么，不需要写代码
组件化预设丰富的组件，覆盖常见场景
可组合组件可以自由组合布局
可交互组件支持用户交互（点击、筛选等）
安全不允许执行任意代码，只能使用预设组件

4.3 Schema 渲染器

// Schema 渲染器
const SchemaRenderer: React.FC<{ schema: ComponentSchema }> = ({ schema }) => {
  const Component = componentRegistry[schema.type];

  if (!Component) {
    console.warn(`Unknown component type: ${schema.type}`);
    return null;
  }

  const children = schema.children?.map((child, index) => (
    <SchemaRenderer key={index} schema={child} />
  ));

  return <Component {...schema.props}>{children}</Component>;
};

4.4 安全校验

Schema 安全校验规则
  1. 检查组件类型是否在白名单中
  2. 检查属性是否包含危险内容（XSS 防护）
  3. 检查 IFrame src 是否在域名白名单中
  4. 递归检查所有子组件
// Schema 安全校验
function validateSchema(schema: ComponentSchema): boolean {
  // 1. 检查组件类型是否在白名单中
  if (!componentRegistry[schema.type]) {
    throw new Error(`Invalid component type: ${schema.type}`);
  }

  // 2. 检查属性是否包含危险内容
  const propsStr = JSON.stringify(schema.props || {});
  if (/<script|javascript:|on\w+=/i.test(propsStr)) {
    throw new Error('XSS attack detected in props');
  }

  // 3. 检查 IFrame src 是否在白名单中
  if (schema.type === 'IFrame') {
    const allowedDomains = ['company.com', 'localhost'];
    const url = new URL(schema.props?.src || '');
    if (!allowedDomains.some(d => url.hostname.endsWith(d))) {
      throw new Error(`IFrame src not allowed: ${url.hostname}`);
    }
  }

  // 4. 递归检查子组件
  schema.children?.forEach(validateSchema);

  return true;
}

5.2 Action 处理器

// Action 处理器
const actionHandlers: Record<string, (payload: any) => void> = {
  // 发送消息到 AI
  chat: (payload) => {
    sendChatMessage(payload.message);
  },

  // 调用 API
  api: async (payload) => {
    const result = await callAPI(payload.endpoint, payload.params);
    // 可以触发 UI 更新
    updateWorkbenchData(result);
  },

  // 导出文件
  export: (payload) => {
    exportData(payload.format, payload.data, payload.filename);
  },

  // 页面导航
  navigate: (payload) => {
    if (payload.external) {
      window.open(payload.url, '_blank');
    } else {
      router.push(payload.path);
    }
  },
};

// 组件中使用
const handleAction = (action: ActionSchema) => {
  const handler = actionHandlers[action.type];
  if (handler) {
    handler(action.payload);
  }
};

// Workbench 数据上下文
interface WorkbenchContext {

// 组件可以读写共享数据
const DataTable: React.FC<DataTableProps> = (props) => {
  const { data, setData } = useWorkbenchContext();

  };

  return (
    <Table
      dataSource={props.dataSource}
      onRow={(record) => ({ onClick: () => handleRowClick(record) })}
      // ...
    />
  );
};

8.2 前端解析

// 解析 AI 回复中的 workbench-schema
function parseWorkbenchSchema(content: string): ComponentSchema | null {
  const match = content.match(/```workbench-schema\n([\s\S]*?)\n```/);
  if (match) {
    try {
      const schema = JSON.parse(match[1]);
      validateSchema(schema);
      return schema;
    } catch (e) {
      console.error('Invalid workbench schema:', e);
    }
  }
  return null;
}

// 在聊天界面中使用
const ChatMessage: React.FC<{ content: string }> = ({ content }) => {
  const schema = parseWorkbenchSchema(content);

  if (schema) {
    // 打开 Workbench 展示
    openWorkbench(schema);
  }

  // 渲染普通文本内容
  return <Markdown>{content.replace(/```workbench-schema[\s\S]*?```/, '')}</Markdown>;
};

9.2 AI Prompt 指导

AI System Prompt 中包含的组件库说明
你可以使用 Workbench 组件库来构建可视化界面。当需要展示数据时，使用以下格式：

```workbench-schema
{JSON Schema}
```

可用组件：
- Container: 容器，props: { title }
- Row/Col: 栅格布局，Col props: { span: 1-24 }
- Statistic: 统计数值，props: { title, value, suffix, trend, trendValue }
- DataTable: 数据表格，props: { columns, dataSource, pagination }
- BarChart/LineChart/PieChart: 图表
- Markdown: Markdown 内容
- CodeBlock: 代码块
- ExportButton: 导出按钮
