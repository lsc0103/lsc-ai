一、设计理念

精美而简约

High-end Minimalism，符合AI平台的现代感、科技感、创新感，去除冗余装饰，保留核心功能美学。

玻璃拟态

Glassmorphism / Liquid Glass，通过磨砂玻璃、半透明层、背景模糊创造深度感和层次感。

空间立体

强调 Z 轴层级、立体透视、浮层深度，而非传统的平面卡片堆叠，营造三维空间感。

核心设计策略

玻璃拟态 + 主题背景 = 一套组件，多种氛围

组件本身采用半透明玻璃效果，背景决定整体氛围。切换主题背景时，背景色"透"过玻璃层，自然形成不同的主题色调。这种架构大幅降低了主题维护成本，无需为每个主题重新定义所有组件颜色。

2.1 高级玻璃卡片

高级玻璃效果

真实的玻璃效果需要多层光学模拟：顶部高光线条（::before）模拟光线在边缘的聚焦反射；渐变边框（::after + mask）模拟光源方向造成的明暗变化；多层 inset shadow 模拟玻璃厚度带来的边缘折射。

2.2 玻璃边缘光线原理

为什么需要边缘光线？

真实世界中的玻璃有厚度，光线穿过玻璃边缘时会发生折射和反射，在边缘处形成明亮的高光线条。平面化的设计缺少这种光学效果，看起来像是"贴图"而非真实材质。

  • 顶部高光：最亮，因为光源通常从上方照射
  • 左侧微光：次亮，模拟左上方光源
  • 底部暗边：形成对比，增强立体感
  • 渐变边框：从亮到暗的过渡，模拟光照方向

三、Z轴层级体系

基于空间深度的层级设计，每层拥有不同的模糊程度和阴影强度，模拟真实的物理深度关系。

4.4 玻璃效果使用场景

效果类型适用场景边缘强度性能消耗
Basic Glass 按钮、小卡片、输入框背景 轻微（1px 高光） 低
Standard Glass 内容卡片、消息气泡、工具提示 标准（1px 高光 + 渐变边框） 中
Thick Glass 主要面板、侧边栏、Modal 强烈（2px 高光 + 粗渐变边框） 中
Glow Glass 重要提示、焦点区域、高亮状态 标准 + 内部发光 中
边缘效果使用原则
  • 越靠近用户的层级（Z值越高），边缘效果越明显
  • Modal、Popover 等浮层使用 Thick Glass，强化"浮起"感
  • 基础控件使用 Basic Glass，避免视觉杂乱
  • 同一视图中，边缘效果强度应形成梯度，而非均匀一致

六、动效规范

6.1 缓动曲线选择

曲线用途物理特性
ease-out-expo常规过渡、hover 反馈快速启动，平滑减速
ease-out-back面板展开、弹窗出现轻微回弹，活泼感
ease-spring拖拽释放、弹性动画弹簧阻尼，物理真实
cubic-bezier(0.4, 0, 0.2, 1)页面切换、长距离移动Material Design 标准

6.2 动画性能规则

推荐做法
  • 只动画 transform 和 opacity
  • 使用 will-change 提示浏览器优化
  • 动画时长控制在 150-400ms
  • 使用 CSS 变量统一管理缓动曲线
  • 复杂动画使用 requestAnimationFrame
避免做法
  • 动画 width、height、padding
  • 动画 top、left（用 transform 代替）
  • 大面积 backdrop-filter 动画
  • 线性动画（缺乏物理感）
  • 花哨的弹跳、旋转动画

七、Apple Vision Pro 设计参考

追求极致 UI 的设计理念
材质真实感

材质应有真实的物理质感，而非简单的贴图效果。玻璃就要有玻璃的通透与反射，金属就要有金属的光泽与重量感。

环境光照

光线来自环境而非组件自身发光。避免组件"自发光"的廉价感，通过阴影和高光暗示统一的光源方向。

惯性与阻尼

动效遵循惯性与阻尼原理，而非线性动画。物体运动应有加速、减速的自然过程，停止时有微妙的回弹。

空间层次

界面元素在三维空间中有明确的前后关系，通过模糊度、透明度、阴影强度传达深度信息。

十一、WebGL/ThreeJS 视觉增强（可选）

定位说明

WebGL/ThreeJS 作为可选的视觉增强技术，用于特定场景的高级视觉效果。核心 UI 组件仍使用纯 CSS 方案，保证性能和兼容性。WebGL 增强功能应设计为渐进增强模式，低端设备自动降级为 CSS 方案。

11.1 技术选型对比

方案适用场景优势劣势
纯 CSS 所有 UI 组件 性能好、兼容性强、体积小 效果上限有限
Three.js 3D 场景、复杂粒子效果 功能完整、生态丰富 体积较大（~150KB gzip）
原生 WebGL/WebGL2 极致性能要求的场景 最小体积、完全控制 开发成本高
GLSL Shader 2D 背景特效 效果惊艳、可控性强 需要 GPU 编程知识

11.2 推荐使用场景

场景效果描述推荐技术优先级
背景动态光晕 流动的渐变光晕、跟随鼠标的光源 GLSL Shader / Three.js 中
首页 Hero 区域 3D 粒子流、液态玻璃变形 Three.js 低
数据可视化 3D 3D 图表、空间数据展示 Three.js / D3 + WebGL 中
加载过渡动画 品牌 Logo 3D 动画 Three.js / Lottie 低
光线折射模拟 真实的玻璃光线追踪效果 WebGL2 + Shader 低（实验性）

11.3 实现架构

// 渐进增强模式架构

interface VisualEnhancement {
  // 检测设备能力
  canUseWebGL: boolean;
  performanceTier: 'high' | 'medium' | 'low';

  // 渐进增强入口
  enhance(element: HTMLElement, effect: EffectType): void;

  // 降级处理
  fallback(element: HTMLElement): void;
}

// 使用示例
const enhancement = new VisualEnhancement();

if (enhancement.canUseWebGL && enhancement.performanceTier === 'high') {
  // 高端设备：启用 WebGL 背景效果
  enhancement.enhance(backgroundElement, 'fluid-gradient');
} else {
  // 低端设备：使用 CSS 静态渐变
  enhancement.fallback(backgroundElement);
}

设计规范总结

本设计规范以iOS 风格玻璃拟态（Glassmorphism）为核心视觉语言，特别强调边缘光线效果的真实感。结合主题背景系统实现"一套组件，多种氛围"的高效架构。通过严格的Z轴层级体系营造空间深度感，遵循物理动效原则保证流畅的60fps交互体验。WebGL/ThreeJS 作为可选的视觉增强技术，采用渐进增强模式，在高端设备上提供更极致的视觉体验。
